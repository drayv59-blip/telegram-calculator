# 🤖 Telegram Calculator Bot

Умный Telegram-бот для расчёта дозы инсулина с автоматическим определением коэффициента по времени суток.

## ✨ Возможности

- 💉 **Расчёт дозы инсулина** по сахару в крови и углеводам
- 🕐 **Автоматическое определение УК** (коэффициента) по времени суток
- ✍️ **Ручной ввод УК** при необходимости
- 🕒 **Ручной ввод времени** - можно указать время расчета
- 💓 **Heartbeat** - периодические сигналы жизни бота (каждый час)
- 🔧 **Health Check** - проверка здоровья приложения для облачных хостингов
- ✅ **Круглосуточная работа** на облачном хостинге (Render, Railway)
- 📊 **Логирование** всех операций в файл и консоль

## 🚀 Быстрый старт

### Локальный запуск

1. **Клонировать репозиторий:**
```bash
git clone https://github.com/drayv59-blip/telegram-calculator.git
cd telegram-calculator
```

2. **Создать виртуальное окружение:**
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

3. **Установить зависимости:**
```bash
pip install -r requirements.txt
```

4. **Создать файл `.env`:**
```bash
cp .env.example .env
```

5. **Заполнить `.env` с твоими данными:**
```bash
TELEGRAM_BOT_TOKEN=твой_токен_от_BotFather
ADMIN_ID=твой_ID_от_userinfobot
PORT=8000
```

6. **Запустить бота:**
```bash
python bot.py
```

## 💉 Примеры использования

### Автоматический УК (по текущему времени МСК)
```
/c 9.3 72
```
- `9.3` - текущий уровень сахара (ммоль/л)
- `72` - количество углеводов (граммы)
- УК определяется автоматически по текущему времени

Результат:
```
🌞 Обед, K1 = 1.33
Доза на еду: 1.98 ед.
ДПС (до 5.5): 1.53 ед.
ИТОГО: 3.5 ед. Фиаспа
```

### Ручной УК
```
/c 9.3 72 1.4
```
- `9.3` - уровень сахара
- `72` - углеводы
- `1.4` - твой коэффициент (УК) вместо автоматического

### Расчет с указанием времени
```
/c 13.1 0 02:30
```
- `13.1` - уровень сахара
- `0` - углеводы (только ДПС без еды)
- `02:30` - время (ночной коэффициент 1.30)

### С запятой вместо точки
```
/c 9,3 72
```

## 💓 Heartbeat (сигнал жизни)

Бот отправляет тебе сообщение **каждый час** (если установлен ADMIN_ID), чтобы подтвердить, что он живой:

```
💓 HEARTBEAT - Бот живой!

🤖 Статус: ✅ Работает
⏰ Время: 15:30:45 МСК
📅 Дата: 22.07.2026
📊 Версия: 2.1
🌍 Зона: Europe/Moscow (UTC+3)
```

**Что делать, если heartbeat не приходит:**
- Проверь, что `ADMIN_ID` правильно установлен
- Посмотри логи приложения
- Перезагрузи сервис

## 🔧 Health Check

HTTP сервер на порту 8000 отвечает `OK` на запросы `/health`, чтобы:
- Облачный хостинг знал, что приложение живо
- Сервис не "засыпал" при отсутствии активности
- Автоматически перезагружался при сбое

| Параметр | Значение | Описание |
|----------|----------|---------|
| Целевой СК | 5.5 ммоль/л | На какой уровень сахара рассчитывается ДПС |

## 📍 Как получить необходимые значения

### Получить TELEGRAM_BOT_TOKEN:
1. Открой Telegram
2. Найди бота `@BotFather`
3. Напиши `/newbot`
4. Следуй инструкциям
5. Скопируй токен вида `123456789:ABCdefGHIjklmnoPQRstuvWXYZ`

### Получить ADMIN_ID:
1. Найди бота `@userinfobot`
2. Напиши ему любое сообщение
3. Он вернёт твой ID (число вида `123456789`)

## 🐛 Решение проблем

### Бот не отвечает
1. Проверь, что `TELEGRAM_BOT_TOKEN` правильный (скопирован полностью без пробелов)
2. Посмотри логи: `tail -f bot.log`
3. Убедись, что интернет подключен
4. Проверь, что бот запущен: `ps aux | grep bot.py`

### Heartbeat не приходит
1. Проверь, что `ADMIN_ID` установлен и правильный
2. Убедись, что это твой реальный Telegram ID
3. Посмотри логи: `grep "Heartbeat" bot.log`
4. Перезагрузи бота: `python bot.py`

### Health Check не работает
1. Проверь, что PORT корректный (обычно 8000)
2. На облачных хостингах PORT устанавливается автоматически
3. Проверь логи на ошибки HTTP сервера

### Ошибка "TELEGRAM_BOT_TOKEN не установлен"
1. Убедись, что создал `.env` файл
2. Проверь, что переменная `TELEGRAM_BOT_TOKEN` заполнена
3. На хостинге добавь переменную в Environment Variables

### Бот зависает или медленно реагирует
1. Проверь интернет соединение
2. Посмотри логи на ошибки
3. Перезагрузи приложение

## 📚 Дополнительная информация

### Как работает расчет
1. Бот получает уровень сахара (СК) и углеводы
2. Определяет текущий час или использует переданное время
3. По часу выбирает коэффициент (K1)
4. Рассчитывает дозу на еду: `(углеводы / 12) × K1`
5. Если СК > 7, рассчитывает ДПС: `(СК - 5.5) / ФЧИ`
6. Складывает обе дозы и округляет до 0.5 единиц
7. Отправляет результат

### Как работает Health Check
- HTTP сервер слушает порт 8000
- На запрос GET `/health` отвечает `OK 200`
- Облачный хостинг проверяет это каждые 5 минут
- Если нет ответа 3 раза подряд - перезагружает приложение

### Как работает Heartbeat
- Асинхронный таск запускается при старте бота
- Каждый час отправляет сообщение администратору
- Если ADMIN_ID не установлен - heartbeat отключен
- Помогает отследить, когда бот завис или упал
